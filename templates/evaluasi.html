Dari setiap 100 URL yang diperiksa sistem, sekitar {{ "%.0f"|format(metrics.accuracy*100) }} URL diprediksi dengan benar. Model ini juga sangat jarang "salah alarm" — ketika model mengatakan sebuah URL adalah phishing, kemungkinan besar ({{ "%.1f"|format(metrics.precision*100) }}%) URL itu memang phishing.
Keterbatasan evaluasi (penting untuk laporan)
  • Evaluasi ini menggunakan pembagian train/test acak baru dari dataset yang tersedia (bukan pembagian identik dengan proses training model asli, karena split original tidak disertakan bersama file model). Hasil tetap representatif karena model dievaluasi pada data yang tidak digunakan untuk fine-tuning pada tahap ini.
  • Akurasi setinggi ini umum terjadi pada dataset PhiUSIIL karena fitur-fiturnya (terutama fitur berbasis konten HTML seperti NoOfExternalRef, LineOfCode) sangat diskriminatif antara situs phishing dan legitimate pada dataset ini.
  • Performa pada URL baru di dunia nyata (khususnya lewat halaman Cek URL) bisa lebih rendah karena keterbatasan ekstraksi fitur real-time.
