Insight & Rekomendasi

Ringkasan temuan utama dari analisis data dan model, disampaikan dalam bahasa yang mudah dipahami untuk mendukung pengambilan keputusan tim keamanan/non-teknis.

🔑 Temuan Utama

01 · HTTPS bukan jaminan mutlak

Hanya {{ stats.https_phish }}% URL phishing pada data menggunakan HTTPS, dibandingkan {{ stats.https_legit }}% pada URL legitimate. Situs tanpa HTTPS jauh lebih layak dicurigai.

02 · Kelengkapan visual mencerminkan legitimasi

Rata-rata jumlah gambar pada halaman phishing ({{ stats.image_phish }}) jauh lebih sedikit dibanding legitimate ({{ stats.image_legit }}). Situs phishing umumnya dibuat terburu-buru hanya untuk meniru tampilan login, tanpa kelengkapan visual situs asli.

03 · Struktur link eksternal jadi pembeda

Rata-rata jumlah link eksternal pada halaman phishing ({{ stats.extref_phish }}) vs legitimate ({{ stats.extref_legit }}) — sesuai dengan NoOfExternalRef yang menjadi fitur paling berpengaruh di model.

04 · Akurasi tinggi, tetap ada batasan

Model sangat akurat pada data historis, dengan F1-Score {{ "%.1f"|format(metrics.f1*100) }}%, tapi tetap memiliki keterbatasan pada pola phishing baru/URL real-time.

📌 Rekomendasi untuk Tim Keamanan

  • Flag otomatis URL yang tidak menggunakan HTTPS dan memiliki jumlah redirect di atas rata-rata — kombinasi ini meningkatkan kecurigaan phishing secara signifikan.
  • Audit berkala terhadap halaman dengan struktur link eksternal tidak wajar.
  • Edukasi pengguna agar tidak hanya mengandalkan ikon gembok (HTTPS) sebagai tanda situs aman.
  • Integrasikan sebagai lapisan tambahan, bukan pengganti sistem keamanan yang sudah ada.
  • Perbarui model secara berkala dengan data URL phishing terbaru.
Dampak Bisnis: Dengan mengintegrasikan sistem deteksi ini ke alur kerja (plugin browser, filter email, atau API internal), organisasi dapat mengurangi risiko kebocoran data, menurunkan biaya insiden keamanan, dan meningkatkan kepercayaan pengguna.
